# 开发者快速上手 — Phase 0 一日入门

> **本文档目标**：让任何一个**第一次接触本项目**的开发者（人 or AI Agent），能在**半小时内**理清"先看什么、再做什么、怎么验证"，避免在 9000 行架构文档里迷路。
>
> **读者画像**：已经会写 TypeScript / 大致用过 Tauri / 第一次看 HSAS。

---

## 〇、TL;DR（30 秒版本）

1. **理论入门**（按顺序读 5 份文档，约 1 小时）：
   - [00-总架构设计.md](./00-总架构设计.md) §〇 + §二 + §十六 → 理解"我们在做什么"
   - [14-通用约定.md](./14-通用约定.md) §〇 → 把 4 个核心术语（Capability / Agent / Skill / ToolPack）先记住
   - [11-HSAS-Spec.md](./11-HSAS-Spec.md) §〇 + §一 + §二 + §三 → 理解 Agent Manifest 长什么样
   - [15-平台底座实现指南.md](./15-平台底座实现指南.md) §〇 → 看一眼 Phase 0 要做的所有文件
   - [13-沙箱实现细节.md](./13-沙箱实现细节.md) §〇（**只看设计原则一节**）→ 知道有这事但 Phase 0 不写

## 三、跑起来：第一次 `pnpm dev`

```bash
# 1. 装依赖
pnpm install

# 2. 跑数据库迁移（如果你已经写完 #8）
pnpm tauri build      # 第一次需要构建 Rust 端

# 3. 开发模式启动
pnpm tauri dev
```

期望看到的控制台输出（`bootstrap()` 完成后）：

```
[platform] Loading builtin manifests...
[platform] Registered 9 builtin capabilities
[platform] Loading user capabilities from SQLite...
[platform] Loaded 0 user capabilities
[platform] Bootstrap done in 47ms
```

如果看到 `BUILT_IN_AGENTS is not defined` 这类报错，说明你忘了改造 [15] §一 的内核（第 13 步）。

## 四、Phase 0 完工验收（一条 SQL + 一次重启）

这是 [15] §十二 末尾给的**金标准验收**——**任何 Phase 0 PR 必须通过这一项**：

```sql
-- 在 ~/.seven-hrops/seven-hrops.db 中执行：
INSERT INTO capabilities (name, kind, manifest_yaml, source, enabled, installed_at)
VALUES (
  'my-test-capability',
  'Capability',
  '<贴一份合法的 Capability YAML，参考 [11] §五 示例>',
  'user',
  1,
  CURRENT_TIMESTAMP
);
```

**验收步骤**：
1. 关闭应用
2. 执行上面那条 SQL
3. 重启应用
4. 打开 ChatWorkspace
5. **能力卡列表多出 1 张「my-test-capability」** ✅

如果第 5 步看不到新卡片，说明 `bootstrap()` 没有从 SQLite 加载用户能力——回去查 [15] §九。

## 五、常见踩坑

| 现象 | 原因 | 解法 |
|---|---|---|
| `tsc` 报 `Cannot find name 'CapabilitySpec'` | 类型分散在多文件 | 统一从 `src/platform/types.ts` 导出 |
| `manifestValidator` 总抛 `INVALID_KIND` | YAML 大小写错了 | `kind` 必须是 `Agent` / `Skill` / `Capability`（首字母大写） |
| Tauri 命令调用返回 `command not allowed` | 没在 `tauri.conf.json` 里登记 | 看 [15] §10.3 末尾的 `allowlist` 配置 |
| 内置能力卡显示 9 张但点进去报错 | seed 时漏了某个 ToolPack | 检查 [12-平台内置工具集.md](./12-平台内置工具集.md) §〇 列出的 10 个 ToolPack 全部 seed |
| 改完代码热更新没生效 | Tauri 的 Rust 代码改动需要重启 | `Ctrl+C` 后重新 `pnpm tauri dev` |
| SQLite 文件位置找不到 | macOS 默认在 `~/Library/Application Support/com.seven-hrops/` | 用 `tauri::api::path::app_data_dir()` 拿绝对路径 |

## 七、和谁结对？（人员分工建议）

如果是 2~3 人并行开发 Phase 0，建议：

- **A（前端 / TS 主力）**：第 1 天 + 第 2~3 天 + 第 5 天的 #11~#14
- **B（Rust 主力 + DB）**：第 4 天 + 第 5 天的 SQL seed
- **C（QA / 单测）**：跟着 A 写 `manifestValidator.test.ts` + 跟着 B 写 Rust 单测

3 人同时开工时，**先约定 `src/platform/types.ts`**（第 1 天 #1），其他人都依赖它。
