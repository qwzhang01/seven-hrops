# 能力 02 — JD 优化（jd-optimization）

> 对应 Agent：`jd-optimizer` | 模式：subagent | 优先级：P1
>
> 📖 工作空间路径、双通道入口、双通道文件录入、静默切换、混合存储原则、共用工具定义 → 见 [14-通用约定.md](./14-通用约定.md)。本文档仅描述本能力特有逻辑。

---

## 一、能力描述

用户提供 JD 文本或文件，Agent 进行合规检测、措辞优化、结构优化，输出优化后的完整 JD，可选保存到 DB 或导出文件。

---

## 二、完整对话流程

```
用户：帮我优化这个 JD（粘贴文本 或 提供文件路径）
  ↓
jd-optimizer Agent：读取 JD 内容（read_file 或直接从对话获取）
  ↓
jd-optimizer Agent：调用 compliance 子 Agent 做合规检测
  ↓
jd-optimizer Agent：生成优化建议（合规/措辞/结构/吸引力）
  ↓
jd-optimizer Agent：输出优化后的完整 JD 全文（流式）
  ↓
用户：可以继续追问修改，或要求保存
  ↓
jd-optimizer Agent：调用 save_jd 保存到 DB（可选）
```

## 七、本能力的差异说明

> 通用部分见 [14-通用约定.md](./14-通用约定.md)。本节仅列出 JD 优化的特殊点。

### 7.1 子目录映射

| 标准位 | 本能力具体名 | 内容 |
|--------|--------------|------|
| `00_inputs/` | `00_inputs/` | 原始 JD 文本 `jd_v0.md` |
| `01_*_raw/` | `01_jd_raw/` | 用户上传的 JD 文件（拖拽来源或手动拷入） |
| `03_*/` | `03_jd_drafts/` | 多轮优化中间版本 `jd_v1.md` `jd_v2.md` ...，便于追溯 |
| `04_reports/` | `04_reports/` | 最终 JD 与合规报告的导出产物 |

### 7.2 本能力专属的存储落点

| 内容 | 存储位置 | 说明 |
|------|---------|------|
| JD 原文 + 优化后版本 | 工作空间 `00_inputs/jd_v0.md` + `03_jd_drafts/jd_v{n}.md` + DB `jds` 表 | 多版本文件便于人工追溯，DB 存当前生效版 |
| 合规检查结果（问题清单） | DB `compliance_results` 表 | 结构化、需要跨 JD 统计与审计 |

### 7.3 报告导出格式选项

JD 优化本身主要产物是文本，可选导出以下形式供外部分享：

| 选项 | 工具 | 阶段 |
|------|------|------|
| Markdown（默认，直接在工作空间中） | `write_file` | Phase 1 |
| HTML（发送候选人报名页） | `export_to_html` | Phase 2 |
| Word（内部评审打印） | `export_to_word` | Phase 2 |
| PPT（招聘需求宣宣） | `export_to_pptx` 配 `hr-default` 模板 | Phase 3 |
