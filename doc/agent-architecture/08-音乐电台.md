# 能力 08 — 音乐电台（music-radio）🎵

> 对应 Agent：`dj` | 模式：subagent | 优先级：P2
> 类型：隐藏能力（不在能力卡片展示，`hidden: true`）
>
> 📖 静默切换机制（`activate_capability` / 触发关键词表）、共用工具分级说明 → 见 [14-通用约定.md](./14-通用约定.md)。本能力**为纯流式能力、不创建工作空间**，所有状态都在 `musicStore` 持久化；本文档仅描述本能力特有逻辑。

---

## 一、能力描述

这是最特殊的 Agent。不再通过关键词硬编码触发，而是由 `dj` Agent 主动感知用户情绪状态，结合当前天气、工作内容、历史歌单偏好，提供个性化音乐推荐，并在音乐前加入个性化解说词。

---

## 二、完整对话流程

```
用户：累了 / 刚搞定一个大需求 / 跟同事吵架了 / 下雨天好适合听歌
  ↓
dj Agent：分析情绪状态（疲惫/兴奋/沮丧/焦虑/平静）
  ↓
dj Agent：调用 get_weather(city) 获取当前天气
  ↓
dj Agent：调用 get_work_context() 获取当前工作内容
  ↓
dj Agent：调用 get_user_playlist() 获取用户历史偏好
  ↓
dj Agent：综合情绪+天气+工作内容+偏好 → 生成推荐理由
  ↓
dj Agent：调用 recommend_tracks(mood, weather, context) 返回推荐歌单
  ↓
dj Agent：生成个性化解说词（100字以内，温暖有趣）
  ↓
dj Agent：调用 update_playlist(tracks) 更新播放器
  ↓
前端：显示播放器 + 解说词 + 歌单
```

### 3.3 `src/stores/musicStore.ts`（新增文件）

```typescript
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

export interface Track {
  id: string;
  title: string;
  artist: string;
  duration: number; // 秒
  tags: string[];
  coverUrl?: string;
  audioUrl: string;
}

export interface AgentRecommendation {
  mood: string;           // 检测到的情绪
  weather: string;        // 当前天气
  djComment: string;      // 解说词文案
  recommendedAt: number;  // 推荐时间戳
}

export interface UserPreferences {
  likedTags: string[];    // 喜欢的音乐标签
  dislikedTags: string[]; // 不喜欢的标签
  playHistory: string[];  // 播放历史（歌曲ID，最近50首）
  recentMoods: Array<{ mood: string; timestamp: number }>; // 最近情绪记录
}

interface MusicStoreState {
  // 播放器状态
  currentTracks: Track[];
  trackIndex: number;
  isPlaying: boolean;
  volume: number;
  isVisible: boolean;

  // Agent 推荐相关
  agentRecommendation: AgentRecommendation | null;

  // 用户偏好（持久化）
  userPreferences: UserPreferences;

  // Actions
  setTracks: (tracks: Track[], djComment?: string, mood?: string) => void;
  setTrackIndex: (index: number) => void;
  setIsPlaying: (playing: boolean) => void;
  setVolume: (volume: number) => void;
  show: () => void;
  hide: () => void;
  toggle: () => void;
  recordPlay: (trackId: string) => void;
  updatePreferences: (prefs: Partial<UserPreferences>) => void;
}

export const useMusicStore = create<MusicStoreState>()(
  persist(
    (set, get) => ({
      currentTracks: [],
      trackIndex: 0,
      isPlaying: false,
      volume: 0.7,
      isVisible: false,
      agentRecommendation: null,
      userPreferences: {
        likedTags: [],
        dislikedTags: [],
        playHistory: [],
        recentMoods: [],
      },

      setTracks: (tracks, djComment, mood) => {
        const now = Date.now();
        set({
          currentTracks: tracks,
          trackIndex: 0,
          isPlaying: true,
          isVisible: true,
          agentRecommendation: djComment ? {
            mood: mood ?? 'calm',
            weather: '',
            djComment,
            recommendedAt: now,
          } : null,
        });
        // 记录情绪历史
        if (mood) {
          const { userPreferences } = get();
          set({
            userPreferences: {
              ...userPreferences,
              recentMoods: [
                { mood, timestamp: now },
                ...userPreferences.recentMoods.slice(0, 19), // 保留最近20条
              ],
            },
          });
        }
      },

      setTrackIndex: (index) => set({ trackIndex: index }),
      setIsPlaying: (playing) => set({ isPlaying: playing }),
      setVolume: (volume) => set({ volume }),
      show: () => set({ isVisible: true }),
      hide: () => set({ isVisible: false }),
      toggle: () => set((s) => ({ isVisible: !s.isVisible })),

      recordPlay: (trackId) => {
        const { userPreferences } = get();
        set({
          userPreferences: {
            ...userPreferences,
            playHistory: [
              trackId,
              ...userPreferences.playHistory.filter((id) => id !== trackId).slice(0, 49),
            ],
          },
        });
      },

      updatePreferences: (prefs) => {
        const { userPreferences } = get();
        set({ userPreferences: { ...userPreferences, ...prefs } });
      },
    }),
    {
      name: 'music-store',
      partialize: (state) => ({
        volume: state.volume,
        userPreferences: state.userPreferences,
      }),
    }
  )
);
```

## 七、本能力的差异说明

> 通用部分见 [14-通用约定.md](./14-通用约定.md)。本能力是**唯一不创建工作空间的内置能力**，以下差异需特别说明。

### 7.1 不适用的通用约定

| 通用约定条目 | 本能力说明 |
|--------------|---------|
| §1.1 路径与命名 / §1.2 标准目录结构 / §1.3 子目录映射 | **不适用**。本能力为流式交互，不产生中间产物文件 |
| §1.4 混合存储策略 | **部分适用**。使用者偏好、歌单、情绪历史存 `musicStore` （LocalStorage 持久化），不入 DB |
| §1.5 双通道文件录入 | **不适用**。本能力无需上传文件 |
| §1.6 右侧资源面板 | **不适用**。右侧面板仅展示 `MiniMusicPlayer.tsx`，不走 `WorkspaceFileTree` |

### 7.2 适用的通用约定

| 通用约定条目 | 本能力使用方式 |
|--------------|---------|
| §2 能力静默切换 | ✅ 完全适用。由于本能力 `hidden: true`，在能力卡中不可见，**唯一入口**依赖 `assistant` 识别“累了 / 烦 / 听点音乐”等关键词 → 调用 `activate_capability('music-radio')` |
| §3.3 导出工具 | ❌ 不适用。本能力无报告输出需求 |

### 7.3 本能力专属的存储落点

| 内容 | 存储位置 | 说明 |
|------|---------|------|
| 用户偏好（likedTags / dislikedTags） | `musicStore` 持久化（LocalStorage） | 本地不上云，体现隐私 |
| 播放历史（最近 50 首） | `musicStore.userPreferences.playHistory` | 仅供推荐去重 |
| 情绪历史（最近 20 条） | `musicStore.userPreferences.recentMoods` | 供 `orchestrator.get_user_emotion_history` 读取 |
| DJ 解说词 | `musicStore.agentRecommendation.djComment` | 只保留当前一句，不做历史 |
| 预置歌单库 | 仓库资源文件 `music-library.json` | 随应用一起发布，不入用户目录 |
