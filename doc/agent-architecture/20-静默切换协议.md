# 静默切换协议（Silent Switch Protocol）

> 版本：v1.0 | 日期：2026-06-01 | 状态：已实施（Phase G）
>
> 对应代码：`src/stores/silentSwitchStore.ts` / `src/components/chat/SilentSwitchToast.tsx` / `src/tool-registry/toolpacks/system-toolpack.ts`

---

## 〇、设计目标

用户在任意能力中输入意图（如"帮我筛简历"），`assistant` Agent 自动路由到目标能力，**全程无需用户手动点击切换**。切换发生后，用户有 3 秒窗口可以撤销（undo），超时后自动确认。

**核心约束**：
- 切换对用户"静默"——不弹确认框，不打断对话流
- 上下文承接——原 session 的最后 5 条消息摘要写入新 session 的 `metadata.transferredFrom`
- 可撤销——3 秒内 undo 可完整恢复原 capability + 原 session

---

## 一、协议流程

```
用户输入 "帮我筛简历"
        ↓
assistant Agent 调用 activate_capability(capability_id="resume-screening")
        ↓
system-toolpack.activate_capability handler:
  1. pauseSession(fromSessionId)          ← 挂起原 session（不销毁）
  2. startSession("screener", {           ← 创建新 session
       transferredFrom: fromSessionId,
       delegateDepth: 0,
     })
  3. transferContext(from, to, {          ← 浅拷贝最后 5 条消息摘要
       lastNMessages: 5
     })
  4. capabilityStore.activateCapability(  ← UI 切换
       "resume-screening"
     )
  5. silentSwitchStore.setPending({       ← 触发 Toast
       fromCapability, fromSessionId,
       toCapability, toSessionId,
     })
        ↓
SilentSwitchToast 出现（3s 倒计时）
        ↓
  ┌─────────────────────────────────────────────┐
  │ 用户点击「撤销」(undo)                        │
  │   → activateCapability(fromCapability)       │
  │   → resumeSession(fromSessionId)             │
  │   → endSession(toSessionId)                  │
  │   → pendingSwitch = null                     │
  └─────────────────────────────────────────────┘
        OR
  ┌─────────────────────────────────────────────┐
  │ 3s 超时（confirm）                            │
  │   → pendingSwitch = null                     │
  │   → 新 session 继续运行                       │
  └─────────────────────────────────────────────┘
```

## 二、状态机

```
                    setPending(info)
  [idle] ──────────────────────────────→ [pending]
                                              │
                          undo()              │  confirm() / 3s timeout
                    ┌─────────────────────────┤
                    ↓                         ↓
                 [idle]                    [idle]
```

### 状态字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `pendingSwitch` | `PendingSwitchInfo \| null` | 当前待确认的切换信息 |
| `countdown` | `number` | 剩余秒数（3, 2, 1, 0） |

### PendingSwitchInfo 结构

```typescript
interface PendingSwitchInfo {
  readonly fromCapability: string   // 切换前的 capabilityId
  readonly fromSessionId: string    // 切换前的 sessionId（已 paused）
  readonly toCapability: string     // 切换后的 capabilityId
  readonly toSessionId: string      // 切换后的 sessionId（active）
}
```

## 三、transferContext 协议

`agentService.transferContext(fromId, toId, { lastNMessages })` 将原 session 的最后 N 条消息浅拷贝为摘要，写入目标 session 的 `metadata.transferredFrom`：

```typescript
// 写入目标 session
toSession.metadata.transferredFrom = {
  sessionId: fromId,
  lastMessages: fromSession.transcripts
    .slice(-lastNMessages)
    .map(m => ({ role: m.role, content: m.content.slice(0, 200) }))
}
```

**约束**：
- 只拷贝最后 `lastNMessages`（默认 5）条，不拷贝完整 transcripts
- 内容截断到 200 字符，避免 context 膨胀
- 不拷贝 tool call 结果（只拷贝 user/assistant 文本消息）

## 四、pause / resume 语义

| 操作 | 前置条件 | 后置状态 | 说明 |
|---|---|---|---|
| `pauseSession(id)` | `state === 'active'` | `state = 'paused'` | 挂起 session，保留 transcripts |
| `resumeSession(id)` | `state === 'paused'` | `state = 'active'` | 恢复 session，可继续对话 |
| `endSession(id)` | 任意 state | `state = 'ended'` | 终止 session，不可恢复 |

**undo 时序**：
1. `activateCapability(fromCapability)` — UI 先切回（用户立刻看到）
2. `resumeSession(fromSessionId)` — 恢复原 session
3. `endSession(toSessionId)` — 清理新 session
4. `pendingSwitch = null` — 清除 Toast

## 五、Toast UX 规范

```
┌─────────────────────────────────────────────────────┐
│  ✓ 已切换到「简历筛选」                               │
│  ████████████░░░░░░░░  2s                            │
│                                    [撤销]            │
└─────────────────────────────────────────────────────┘
```

- 进度条从满到空，3 秒后消失
- 「撤销」按钮：`data-testid="silent-switch-undo"`
- Toast 位置：屏幕右下角，z-index 最高
- 切换时输入框禁用 100ms（防止用户在切换瞬间误输入）

## 六、与 Delegate 协议的区别

| 维度 | 静默切换（activate_capability） | Delegate（delegate_to_subagent） |
|---|---|---|
| 触发方 | assistant Agent | 任意 Agent |
| 用户感知 | 有 Toast + undo | 有 DelegateBubble（嵌入式） |
| session 关系 | 原 session paused，新 session active | 父 session active，子 session active |
| 深度限制 | 无（capability 切换是平级的） | 最大深度 1（depth ≥ 2 抛 DelegateNestingForbidden） |
| 跨 source | 不适用（capability 切换不涉及 source） | 禁止（builtin → user 抛 DelegateCrossSourceForbidden） |
| undo 支持 | ✅ 3s 窗口 | ❌ 不支持 undo |

---

## 七、错误码

| 错误码 | 触发条件 | 处理方式 |
|---|---|---|
| `CAPABILITY_NOT_FOUND` | `activate_capability` 传入不存在的 capabilityId | 返回错误，不切换 |
| `CAPABILITY_HIDDEN` | 目标 capability 的 `hidden: true` | 返回错误，不切换 |
| `DELEGATE_NESTING_FORBIDDEN` | `delegate_to_subagent` 时 `delegateDepth >= 2` | 抛错，父 session 继续 |
| `DELEGATE_CROSS_SOURCE_FORBIDDEN` | `delegate_to_subagent` 时 parent.source ≠ target.source | 抛错，父 session 继续 |
