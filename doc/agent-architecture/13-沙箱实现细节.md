# 沙箱实现细节 — Sandbox v1

> 版本：v1.0 | 日期：2026-05-27 | 状态：设计稿
>
> 把 [00-总架构设计.md](./00-总架构设计.md) §4.4 / §14 中规划的"用户 Agent 沙箱"落到具体可执行的代码与拦截点。本文目标是**让用户写出来的 Agent 即便完全胡来，也炸不掉系统**。

---

## 〇、设计原则

1. **应用级沙箱**：在 SessionProcessor / ToolRuntime / Tauri Command 三个拦截点设置 guard，不引入 docker / wasm 等重型方案
2. **Default Deny**：默认拒绝；任何越界操作必须有显式声明（manifest）+ 显式授权（用户确认）
3. **可观测**：所有越界尝试落审计日志，便于事后追溯
4. **可演进**：v1 仅做应用级 guard；v2.0 marketplace 时再叠加签名校验和资源配额硬隔离

## 一、沙箱拦截点全景

```
┌─────────────────────────────────────────────────────────────────┐
│  用户 Agent 一次工具调用的生命周期                                │
└─────────────────────────────────────────────────────────────────┘

  Agent 输出 tool_use(name=write_file, args={path,content})
        │
        ▼
  ┌─────────────────────────┐
  │  ① SessionProcessor      │  ← 拦截点 1：tool 是否在 manifest 白名单
  │     toolWhitelistGuard   │     工具名是否符合 source 默认权限
  └─────────────────────────┘
        │  pass
        ▼
  ┌─────────────────────────┐
  │  ② Permission.Service    │  ← 已有：弹窗 ask / 黑白名单
  └─────────────────────────┘
        │  pass
        ▼
  ┌─────────────────────────┐
  │  ③ rateLimiter           │  ← 拦截点 2：每分钟工具调用次数 / 单会话 token
  └─────────────────────────┘
        │  pass
        ▼
  ┌─────────────────────────┐
  │  ④ ToolRuntime invoke    │  ← 调用 MCP 工具实现
  └─────────────────────────┘
        │
        ▼
  ┌─────────────────────────┐
  │  ⑤ Tauri Command         │  ← 拦截点 3：path/host 白名单（Rust 端）
  │     fsGuard / netGuard   │
  └─────────────────────────┘
        │  pass
        ▼
  执行
```

## 三、拦截点 1：ToolWhitelistGuard

挂载到 `SessionProcessor` 的 tool dispatch 之前：

```typescript
// src/platform/sandbox/toolWhitelistGuard.ts
import { TOOL_REGISTRY } from "../toolRegistry"
import type { AgentManifest } from "../manifestSchema"

export class ToolWhitelistGuard {
  /** Loader 在加载 Agent 时调用一次，把 manifest 存起来 */
  private map = new Map<string, AgentManifest>()
  register(agent: AgentManifest) { this.map.set(agent.metadata.name, agent) }
  unregister(name: string) { this.map.delete(name) }

  /** SessionProcessor 每次 dispatch tool 调用前检查 */
  check(agentName: string, toolName: string): { ok: true } | { ok: false; reason: string } {
    const m = this.map.get(agentName)
    if (!m) return { ok: false, reason: "AGENT_NOT_REGISTERED" }

    // 1) deny 名单优先
    if (m.spec.tools.deny?.includes(toolName)) {
      return { ok: false, reason: "DENIED_BY_MANIFEST" }
    }
    // 2) 必须在 allowed 名单
    if (!m.spec.tools.allowed.includes(toolName)) {
      return { ok: false, reason: "NOT_IN_ALLOWED" }
    }
    // 3) ToolRegistry 校验 source
    const meta = TOOL_REGISTRY[toolName]
    if (!meta) return { ok: false, reason: "UNKNOWN_TOOL" }
    if (!meta.defaultAllowedSources.includes(m.metadata.source)) {
      return { ok: false, reason: "TOOL_NOT_PERMITTED_FOR_SOURCE" }
    }
    return { ok: true }
  }
}

export const toolWhitelistGuard = new ToolWhitelistGuard()
```

挂载位置（伪代码）：

```typescript
// src/agent-runtime/session/processor.ts
async function dispatchTool(agentName: string, toolName: string, args: any) {
  const r = toolWhitelistGuard.check(agentName, toolName)
  if (!r.ok) {
    emitEvent("tool-blocked", { agentName, toolName, reason: r.reason })
    throw new Error(`SANDBOX_BLOCKED: ${r.reason}`)
  }
  // ... 原有逻辑
}
```

## 四、拦截点 2：RateLimiter + TokenQuota

```typescript
// src/platform/sandbox/rateLimiter.ts

interface SessionState {
  tokens: number
  toolCalls: { timestamp: number }[]
  startedAt: number
}

export class RateLimiter {
  private states = new Map<string, SessionState>()  // key: sessionID

  start(sessionID: string) {
    this.states.set(sessionID, { tokens: 0, toolCalls: [], startedAt: Date.now() })
  }

  end(sessionID: string) {
    this.states.delete(sessionID)
  }

  /** 工具调用前检查 */
  checkToolCall(sessionID: string, cfg: SandboxConfig): boolean {
    const s = this.states.get(sessionID)
    if (!s) return true
    const now = Date.now()
    s.toolCalls = s.toolCalls.filter((t) => now - t.timestamp < 60_000)
    if (s.toolCalls.length >= cfg.maxToolCallsPerMinute) return false
    s.toolCalls.push({ timestamp: now })
    return true
  }

  /** LLM 流式 chunk 时累计 token */
  addTokens(sessionID: string, n: number) {
    const s = this.states.get(sessionID)
    if (s) s.tokens += n
  }

  /** 检查 token 配额 */
  checkTokenQuota(sessionID: string, cfg: SandboxConfig): boolean {
    const s = this.states.get(sessionID)
    if (!s) return true
    return s.tokens < cfg.maxTokensPerSession
  }
}

export const rateLimiter = new RateLimiter()
```

挂载到 `SessionProcessor`：
- `session-start` → `rateLimiter.start(sessionID)`
- `tool-start` → `rateLimiter.checkToolCall()`，false 时抛 `RATE_LIMIT_EXCEEDED`
- `text-delta` → `rateLimiter.addTokens(sessionID, chunk.length / 4)`（粗估）
- 每次循环检查 `rateLimiter.checkTokenQuota`，false 时暂停并询问"已使用 100K Token，是否继续？"

```rust
use std::sync::Mutex;
use once_cell::sync::Lazy;

#[derive(Clone, Serialize, Deserialize, Debug)]
pub struct SandboxContext {
    pub agent_name: String,
    pub source: String,                  // builtin | user | marketplace
    pub fs_allowed_read: Vec<String>,    // 已经渲染过模板变量的具体路径
    pub fs_allowed_write: Vec<String>,
    pub network_allowed_hosts: Vec<String>,

/// 当前会话的沙箱上下文（per session 级别）
static SANDBOX_MAP: Lazy<Mutex<std::collections::HashMap<String, SandboxContext>>> =
    Lazy::new(|| Mutex::new(std::collections::HashMap::new()));

#[tauri::command]
pub fn sandbox_set_context(session_id: String, ctx: SandboxContext) {
    SANDBOX_MAP.lock().unwrap().insert(session_id, ctx);
}

#[tauri::command]
pub fn sandbox_clear_context(session_id: String) {
    SANDBOX_MAP.lock().unwrap().remove(&session_id);
}

pub fn get_context(session_id: &str) -> Option<SandboxContext> {
    SANDBOX_MAP.lock().unwrap().get(session_id).cloned()
}
```

### 5.2 文件系统 Guard

```rust
// src-tauri/src/sandbox/fs_guard.rs

use std::path::Path;
use globset::{Glob, GlobSet, GlobSetBuilder};

pub fn check_read(session_id: &str, abs_path: &Path) -> Result<(), String> {
    let ctx = match get_context(session_id) {
        Some(c) => c,
        None => return Err("SANDBOX_NO_CONTEXT".into()),
    };
    if ctx.source == "builtin" {
        return Ok(());
    }
    if !match_any(&ctx.fs_allowed_read, abs_path) {
        return Err(format!("FS_READ_DENIED: {}", abs_path.display()));
    }
    Ok(())
}

pub fn check_write(session_id: &str, abs_path: &Path) -> Result<(), String> {
    let ctx = match get_context(session_id) {
        Some(c) => c,
        None => return Err("SANDBOX_NO_CONTEXT".into()),
    };
    if ctx.source == "builtin" {
        return Ok(());
    }
    if !match_any(&ctx.fs_allowed_write, abs_path) {
        return Err(format!("FS_WRITE_DENIED: {}", abs_path.display()));
    }
    Ok(())
}

fn match_any(globs: &[String], path: &Path) -> bool {
    let mut builder = GlobSetBuilder::new();
    for g in globs {
        let expanded = expand_home(g);
        if let Ok(glob) = Glob::new(&expanded) {
            builder.add(glob);
        }
    }
    let set = match builder.build() {
        Ok(s) => s,
        Err(_) => return false,
    };
    set.is_match(path)
}

fn expand_home(p: &str) -> String {
    if p.starts_with("~") {
        if let Some(home) = dirs::home_dir() {
            return p.replacen("~", home.to_str().unwrap_or(""), 1);
        }
    }
    p.to_string()
}
```

修改所有文件 IO 命令在最前面接入：

```rust
// src-tauri/src/commands/file_io.rs

pub async fn write_text_file(session_id: String, path: String, content: String) -> Result<(), String> {
    let abs = std::fs::canonicalize(&path).map_err(|e| e.to_string())?;
    crate::sandbox::fs_guard::check_write(&session_id, &abs)?;
```

### 5.3 网络 Guard

> Tauri 默认不让前端直接 fetch 任意 host，但 MCP 工具内部可能用 fetch。需要在工具实现层加 guard。

```typescript
// src/platform/sandbox/networkGuard.ts
export function networkGuard(sessionID: string, url: string) {
  const ctx = getSandboxContext(sessionID)  // 前端缓存的副本
  if (!ctx || ctx.source === "builtin") return
  const host = new URL(url).hostname
  const allowed = ctx.networkAllowedHosts
  if (!allowed.includes(host) && !allowed.includes("*")) {
    throw new Error(`NETWORK_DENIED: ${host} not in allowedHosts=${allowed.join(",")}`)
  }
}

// 用法：MCP 工具内部所有 fetch 前调用
async function fetchWithGuard(sessionID: string, url: string, init?: RequestInit) {
  networkGuard(sessionID, url)
  return fetch(url, init)
}
```

## 六、Manifest 静态分析（dryRun）

`agent-builder.install_capability` 写库前的最后一道防线，做"安装前体检"：

```typescript
// src/platform/sandbox/dryRun.ts
import type { AgentManifest } from "../manifestSchema"

export interface DryRunIssue {
  level: "error" | "warning"
  code: string
  message: string
}

const PROMPT_REDFLAGS = [
  /忽略.{0,5}(?:之前|以上|先前)/,
  /ignore previous instructions/i,
  /you are now/i,
  /system prompt:/i,
]

const PATH_BLACKLIST = [
  /\/\.\./,           // 路径穿越
  /^\/etc\//,
  /^\/System\//,
  /^C:\\Windows\\/i,
]

export function dryRun(manifest: AgentManifest): DryRunIssue[] {
  const issues: DryRunIssue[] = []

  // 1) Prompt 越权指令检测
  for (const re of PROMPT_REDFLAGS) {
    if (re.test(manifest.spec.basePrompt)) {
      issues.push({
        level: "warning",
        code: "PROMPT_SUSPICIOUS_INSTRUCTION",
        message: `basePrompt 含可疑越权指令模式：${re.source}`,
      })
    }
  }

  // 2) 路径黑名单检测
  const paths = [
    ...(manifest.spec.filesystem?.readPaths ?? []),
    ...(manifest.spec.filesystem?.writePaths ?? []),
  ]
  for (const p of paths) {
    for (const re of PATH_BLACKLIST) {
      if (re.test(p)) {
        issues.push({
          level: "error",
          code: "PATH_BLACKLISTED",
          message: `路径在系统黑名单中：${p}`,
        })
      }
    }
  }

  // 3) network.allowedHosts 黑名单
  for (const h of manifest.spec.network?.allowedHosts ?? []) {
    if (h === "*") {
      issues.push({
        level: "error",
        code: "NETWORK_WILDCARD_NOT_ALLOWED",
        message: "用户/marketplace Agent 不允许通配 *",
      })
    }
  }

  // 4) 工具组合风险（敏感工具 + 网络）
  const hasNetwork = (manifest.spec.network?.allowedHosts?.length ?? 0) > 0
  const hasSensitive = manifest.spec.tools.allowed.some((t) =>
    ["send_wecom_message", "transcribe_audio"].includes(t),
  )
  if (hasNetwork && hasSensitive) {
    issues.push({
      level: "warning",
      code: "RISKY_COMBINATION",
      message: "同时启用网络 + 敏感工具，请确认必要性",
    })
  }

  return issues
}
```

`install_capability` 工具内部：

```typescript
const issues = dryRun(manifest)
if (issues.some((i) => i.level === "error")) {
  return { installed: false, issues }
}
// 把 warning 渲染到 ManifestPreview，等用户确认
```

## 七、审计日志

```rust
// src-tauri/src/sandbox/audit.rs

use serde::Serialize;
use std::path::PathBuf;
use chrono::Utc;

#[derive(Serialize)]
struct AuditEntry {
    timestamp: String,
    event: String,         // fs_read | fs_write | net_request | sandbox_block ...
    session_id: String,
    detail: String,
}

pub fn audit_log(event: &str, session_id: &str, detail: &str) -> Result<(), String> {
    let entry = AuditEntry {
        timestamp: Utc::now().to_rfc3339(),
        event: event.to_string(),
        session_id: session_id.to_string(),
        detail: detail.to_string(),
    };
    let line = serde_json::to_string(&entry).map_err(|e| e.to_string())?;
    let log_path: PathBuf = dirs::home_dir()
        .ok_or("no home")?
        .join(".seven-hrops")
        .join("audit")
        .join(format!("{}.jsonl", Utc::now().format("%Y-%m-%d")));
    std::fs::create_dir_all(log_path.parent().unwrap()).map_err(|e| e.to_string())?;
    use std::io::Write;
    let mut f = std::fs::OpenOptions::new()
        .create(true)
        .append(true)
        .open(log_path)
        .map_err(|e| e.to_string())?;
    writeln!(f, "{}", line).map_err(|e| e.to_string())?;
    Ok(())
}
```

事件粒度（v1 必落）：
- `sandbox_block`：被任一 guard 拦截
- `fs_read` / `fs_write`：所有 user/marketplace Agent 的文件 IO（builtin 不记，避免日志爆炸）
- `net_request`：所有外部 fetch
- `tool_call`：所有 sensitive 类工具调用

CapabilityManager 的"详情"页可以展示某 Agent 的最近 100 条审计日志，用于排查与举证。

## 九、典型违规场景与处置

| 场景 | 拦截层 | 用户感知 | 审计动作 |
|---|---|---|---|
| 用户 Agent 调 `send_wecom_message` | ToolWhitelistGuard | 报错"工具未授权" | 记 `sandbox_block` |
| 用户 Agent 写到 `/etc/passwd` | Rust fs_guard | Tauri 命令失败 | 记 `sandbox_block` + `fs_write` |
| 用户 Agent fetch 任意外网 | networkGuard | 报错"NETWORK_DENIED" | 记 `sandbox_block` + `net_request` |
| 单会话 Token 用完 | RateLimiter | 暂停 + 询问"已用 100K Token，是否继续？" | 不记（属正常配额） |
| 1 分钟内 100 次工具调用 | RateLimiter | 等下一分钟自动恢复 + Toast | 记 `sandbox_block` |
| Manifest 含越权 prompt | dryRun | 安装前预览页打 ⚠️ 告警 | 记 `dry_run_warning` |

## 十一、和现有模块的交互

| 模块 | 接入点 |
|---|---|
| `Permission.Service` | 沙箱不替代 Permission；Permission 决定"用户是否同意"，沙箱决定"代码层面是否能做" |
| `SessionProcessor` | 在 tool dispatch 之前插 ToolWhitelistGuard + RateLimiter |
| `ToolRuntime` | MCP 工具内部的 fetch 通通走 fetchWithGuard |
| `CapabilityRegistry` | install/uninstall 时同步刷新 toolWhitelistGuard |
| Tauri 文件命令 | 全部接收 sessionID，调用 fs_guard |
