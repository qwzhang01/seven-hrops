# Seven HROps — 重构实施计划（Clean-Slate Plan）

> 版本：v1.0 | 日期：2026-05-28 | 状态：待评审
>
> **基本原则（强制）**：
> 1. **不做任何向后兼容**——所有"绕过 Agent Runtime / Platform Layer"的旧代码（v2 UI 中的 Skill 字面量组件、`intentService`、重复的 `mcp-server`、字面量 `config/capabilities.ts`）一律物理删除。
> 2. **高内聚、低耦合**——每一层只做一件事，跨层禁止直接 import；所有跨层调用必须通过明确的 Service 接口。
> 3. **Agent Runtime 是唯一执行通路**——任何"用户输入 → AI 输出"的链路必须经过 `Session / SessionProcessor / Tool 执行器`，UI 层不允许内嵌业务逻辑。
> 4. **Manifest 是唯一可信源**——内置和用户能力共用同一套 `~/.seven-hrops/manifests/` + SQLite 双源，禁止字面量配置。
> 5. **TS / Rust 边界明确**——业务编排和 UI 走 TS；安全沙箱、文件 IO、解析、转写走 Rust（详见各阶段）。
>

## 二、Phase B — 平台底座重塑（L0/L1/L2/L3 同步收敛）

> **目标**：清场后第一步是把"地基"立起来。底座不依赖任何业务能力，能跑空也能跑。

### B.1 L0 Rust 底座（最小集，沙箱前置）

| 模块 | 路径 | 交付 |
|---|---|---|
| `sandbox` | `src-tauri/src/sandbox/{mod,fs_guard,audit}.rs` | `SandboxContext` per session、`check_read/check_write`、JSON Lines 审计日志（参考 13 号文档） |
| `fs_native` | `src-tauri/src/native/fs.rs` | `read_text_file` / `write_text_file` / `list_dir_entries` / `stat_file` / `canonicalize_path`，**全部接收 `session_id` 参数** |
| `parse_native` | `src-tauri/src/native/parse.rs` | `parse_pdf` / `parse_docx` / `parse_excel` 通用接口（淘汰原 `parse_resume_batch`） |
| `webserver_native` | `src-tauri/src/native/webserver.rs` | `webserver_publish(content, kind)`，axum 实现，返回 `{url, token, qr_png_b64}`（替代 `preview_server.rs`） |
| `transcribe_native` | `src-tauri/src/native/transcribe.rs` | `transcribe_audio(path, lang?)`，whisper.cpp FFI（暂可 stub，Phase 5 落地实现） |

> **注意**：`email/` 目录保留，但所有 Tauri 命令签名加上 `session_id`，由 sandbox 校验路径白名单。

### B.2 L1 Tool Registry 单一入口

```

├── index.ts                # 只做：导入 toolpacks + 把 ToolRegistry 元数据注册到 MCP
├── toolpacks/
```

## 三、Phase C — 服务层与 Store 层重建（L4/L5）

> **目标**：把"业务编排"从 UI 拽出来，统一塞进 services/，UI 只通过 stores 看到状态。

### C.1 services/ 最终形态

| 文件 | 职责 |
|---|---|
| `agentService.ts` | 桥接 UI → Agent Runtime；流式回调（onTextDelta / onToolStart / onToolComplete / onFinish） |
| `capabilitySession.ts` | `getOrCreate(capabilityId)` / `endAll()`；session 复用键 = capabilityId + workspaceId |
| `contextBuilder.ts` | `{{var}}` 模板渲染；contextKeys 注入 |
| `workspaceManager.ts` | 创建工作空间 / 列举 / 元信息读写（调 fs-toolpack） |
| `orchestratorBridge.ts` | 企微 webhook 入站 → orchestrator session；出站推送 |
| `agentBuilderService.ts` | 自然语言 → 调 agent-builder Agent；产出预览 + 安装 |
| `metricsService.ts` | 工具调用 / token / 错误码上报（落到 audit log） |

**严禁**：
- ❌ services 直接调 `invoke()`（必须经 mcp-server tool 或 platform 暴露的 API）
- ❌ services 持有 React state（用 stores）

### C.2 stores/ 最终形态

| Store | 职责 | 数据来源 |
|---|---|---|
| `aiStore.ts` | 当前会话消息 / typing 状态 | agentService 流式回调 |

| `workspaceStore.ts` | 当前工作空间路径 + 文件树（合并 workspaceFileStore） | `workspaceManager` |
| `taskStore.ts` | 全局任务 | orchestrator 推送 |
| `musicStore.ts` ★新建 | 当前播放 / 心情标签 / DJ 解说词 | dj agent 工具产物 |
| `permissionStore.ts` ★新建 | Permission 弹窗队列 | Permission.Service 事件 |
| `layoutStore.ts` | UI 布局 | 本地 |

### C.3 验收

- [ ] 任意 UI 组件不出现 `await invoke(`、不出现 `agentService` 之外的 runtime 调用
- [ ] services 单测覆盖率 ≥ 80%
