# 能力 04 — 笔试准备（written-test）

> 对应 Agent：`exam-designer` | 模式：subagent | 优先级：P1
>
> 📖 工作空间路径、双通道入口、双通道文件录入、静默切换、混合存储原则、`export_to_html/word` 定义 → 见 [14-通用约定.md](./14-通用约定.md)。本文档仅描述本能力特有逻辑。

---

## 一、能力描述

用户提供 JD 和考察方向，Agent 生成结构化笔试题（含答案和评分标准），支持导出 Word 或生成可在本地浏览器作答的交互式 HTML 页面。

---

## 二、完整对话流程

```
用户：帮我生成笔试题（提供 JD 和考察方向）
  ↓
exam-designer Agent：读取 JD，分析岗位技能要求
  ↓
exam-designer Agent：询问题目类型偏好（选择/编程/案例等）和数量
  ↓
exam-designer Agent：生成题目（含答案和评分标准）
  ↓
流式输出题目内容
  ↓
用户：可以要求调整，或要求导出
  ↓
exam-designer Agent：调用 export_to_html（interactive=true）生成可作答页面
                  或 调用 export_to_word 生成 Word 版本
```

## 八、本能力的差异说明

> 通用部分见 [14-通用约定.md](./14-通用约定.md)。本节仅列出笔试准备的特殊点。

### 8.1 子目录映射

| 标准位 | 本能力具体名 | 内容 |
|--------|--------------|------|
| `00_inputs/` | `00_inputs/` | 出题要求、考察重点、题型分布设定 |
| `01_*_raw/` | `01_jd_refs/` | JD 文本快照（出题依据） |
| `03_*/` | `03_question_bank/` | 题目草稿、多版本调整；可后续累积为岗位题库 |
| `04_reports/` | `04_reports/` | `exam.md`（面试官版含答案）、`exam.html`（候选人作答页）、`exam.docx` |

### 8.2 本能力专属的存储落点

| 内容 | 存储位置 | 说明 |
|------|---------|------|
| 题目 + 参考答案 + 评分要点 | DB `exam_papers` 表 + 工作空间 `04_reports/exam.md` | DB 为可信源，文件供人工查看 |
| 候选人答题记录（交互式 HTML 提交） | DB `exam_submissions` 表 | 需要跨候选人对比、跨题统计正确率 |

### 8.3 报告导出格式选项

本能力出题后需要两份产物：

| 选项 | 工具 | 说明 | 阶段 |
|------|------|------|------|
| Word（面试官版，含答案与评分要点） | `export_to_word` | 打印给面试官 | Phase 2 |
| 交互式 HTML（候选人版，不含答案、含计时器） | `export_to_html({ interactive: true })` | 发送链接给候选人作答 | Phase 2 |
| PPT（面试总结讨论题型分布） | `export_to_pptx` 配 `interview` 模板 | 体系化招聘项目可选 | Phase 3 |

> “交互式 HTML 作答页”是本能力只有的产物形态，详细页面规格见 §五。

```yaml
  basePrompt: |
    你是专业的笔试题设计师，为 HR 团队设计岗位笔试题。

    支持题型：单选 / 多选 / 简答 / 编程 / 案例分析

    - 题号和类型
    - 题目内容
    - 选项（选择题）
    - 参考答案
    - 评分标准 / 评分要点

    含计时器 / 单选多选输入 / 简答 textarea / 编程题代码框 / 提交后显示得分。
  contextTemplate: |
    当前工作空间：{{workspacePath}}
    JD ID：{{jdId}}
  contextKeys: [workspacePath, jdId]
  skills:
    - exam-design
  tools:
    allowed:
      - get_jd
      - read_file
      - export_to_html
      - export_to_word
    autoApprove:
      - get_jd
      - read_file
  permission:
    - { permission: read,   pattern: "*", action: allow }
    - { permission: export, pattern: "*", action: allow }
  model:
    provider: anthropic
    modelID: claude-4.7-sonnet
    temperature: 0.6
  capabilityBinding:
    capabilityId: written-test
    autoCreate: false
```
