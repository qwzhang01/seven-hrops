# 能力 05 — 面试评价（interview-eval）

> 对应 Agent：`interview-evaluator` | 模式：subagent | 优先级：P1
>
> 📖 工作空间路径、双通道入口、双通道文件录入（包含音频拖拽上传）、静默切换、混合存储原则、`transcribe_audio/export_to_*` 定义 → 见 [14-通用约定.md](./14-通用约定.md)。本文档仅描述本能力特有逻辑。

---

## 一、能力描述

用户上传面试录音/视频或粘贴面试记录文本，Agent 结合 JD、简历、笔试结果、面试记录，生成综合面试评价报告，支持保存到 DB。

---

## 二、完整对话流程

```
用户：帮我生成面试评价（提供候选人 ID 或上传面试记录）
  ↓
interview-evaluator Agent：读取 JD + 简历 + 历史筛选结果
  ↓
如果是音频/视频文件：
  interview-evaluator Agent：调用 transcribe_audio 转文字
  ↓
interview-evaluator Agent：综合分析所有信息
  ↓
interview-evaluator Agent：生成综合评价报告（5个维度）
  ↓
流式输出报告内容
  ↓
interview-evaluator Agent：调用 save_interview_eval 保存到 DB
```

## 七、本能力的差异说明

> 通用部分见 [14-通用约定.md](./14-通用约定.md)。本节仅列出面试评价的特殊点。

### 7.1 子目录映射

| 标准位 | 本能力具体名 | 内容 |
|--------|--------------|------|
| `00_inputs/` | `00_inputs/` | 面试背景说明、补充上下文 |
| `01_*_raw/` | `01_audio_raw/` | 面试原始录音 / 录像（mp3 / mp4 / wav / m4a） |
| `02_*_parsed/` | `02_transcripts/` | Whisper 转写的面试逐字记录（txt + 含时间戳的 srt） |
| `03_*/` | `03_evaluation/` | 各维度评分调整中间产物 |
| `04_reports/` | `04_reports/` | 评价报告 `eval.md` + HTML/Word/PPT 导出 |

### 7.2 本能力专属的存储落点

| 内容 | 存储位置 | 说明 |
|------|---------|------|
| 5 维度评分 + 推荐结论 + 亮点/风险/入职建议 | **仅** DB `interview_evaluations` 表 | 结构化，需排序/跨候选人对比，单一可信源 |
| 面试转写原文（逐字） | 工作空间 `02_transcripts/{interviewId}.txt` | 体积大、需人工检阅，不进 DB |
| 转写的句子级引用（评分依据） | DB `interview_evaluations.dimensions` JSON 字段内嵌 | 评分与依据原文强绑定，作为审计证据 |
| 报告 Markdown 主体 | DB `interview_evaluations.full_report` + 工作空间 `04_reports/eval.md` | 双向同步 |

### 7.3 报告导出格式选项

报告生成后，interview-evaluator Agent 询问导出格式：

| 选项 | 工具 | 阶段 | 使用场景 |
|------|------|------|---------|
| HTML（默认，含评分雷达图） | `export_to_html` | Phase 2 | 业务负责人、用人部门在线评阅 |
| Word | `export_to_word` | Phase 2 | 入职归档、纸质存档 |
| PPT（面试委员会汇报） | `export_to_pptx` 配 `interview` 模板 | Phase 3 | 高 P 岗位面试委员会汇报决议 |
| 暂不导出 | — | — | 仅存入 DB |
